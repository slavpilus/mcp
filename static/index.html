üîß Available Tools

get_order_status

Get status for a specific order

Parameters:

  • order_id (string) - The order identifier (e.g., "ORD-1001")
  • customer_id (string, optional) - Customer identifier for validation

cancel_order

Cancel an order

Parameters:

  • order_id (string) - The order identifier to cancel
  • reason (string, optional) - Reason for cancellation
  • customer_id (string, optional) - Customer identifier for validation

process_return

Process a return request for an order

Parameters:

  • order_id (string) - The order identifier to return
  • item_ids (array, optional) - List of specific item IDs to return
  • reason (string, optional) - Reason for return
  • customer_id (string, optional) - Customer identifier for validation

track_package

Track package delivery status for an order

Parameters:

  • order_id (string) - The order identifier to track
  • customer_id (string, optional) - Customer identifier for validation

get_support_info

Get customer support information for a specific topic

Parameters:

  • topic (string, optional) - Support topic: "returns", "shipping", "contact", or "general"
  • customer_id (string, optional) - Customer identifier

üß™ Dynamic Testing Data

Enneagora features a dynamic mock data system where order behavior is determined by the order ID pattern. You can test various scenarios using these special order IDs:

Order Status Patterns

Use these order ID suffixes to test different order states:

  • ORD-XXXX-D - Delivered orders (e.g., ORD-1001-D)
  • ORD-XXXX-S - Shipped orders with tracking (e.g., ORD-1002-S)
  • ORD-XXXX-T - In Transit orders with location updates (e.g., ORD-1003-T)
  • ORD-XXXX-P - Processing orders (e.g., ORD-1004-P)
  • ORD-XXXX-R - Ready for Pickup (e.g., ORD-1005-R)
  • ORD-XXXX-C - Cancelled orders (e.g., ORD-1006-C)
  • ORD-XXXX-F - Failed/Problem orders (e.g., ORD-1007-F)
  • ORD-XXXX-E - Error/Not Found (e.g., ORD-1008-E)
  • ORD-XXXX - Pending orders (no suffix, e.g., ORD-1009)

Special Test Cases

Try these specific scenarios:

  • ORD-ERROR - Always returns "not found"
  • ORD-1001-F - Failed order with "lost package" tracking
  • ORD-2002-F - Cannot be cancelled (simulates payment issues)
  • ORD-3003-D - Delivered order with full tracking history

üìö Example Usage

Once connected, you can ask Claude:

  • "Check the status of order ORD-1001-S" (Shipped order)
  • "Cancel order ORD-1004-P" (Processing order - should work)
  • "Cancel order ORD-1002-D" (Delivered order - should fail)
  • "Track order ORD-1003-T" (In transit with location)
  • "Check order ORD-1005-R" (Ready for pickup)
  • "What happened to order ORD-1007-F?" (Failed order)
  • "Find order ORD-1008-E" (Error case)
  • "What's your return policy?"
