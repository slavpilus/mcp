üñ•Ô∏è Claude Desktop (Local Server)

For Claude Desktop, you need to run the server locally and configure it for STDIO transport:

Step 1: Clone and Setup

git clone https://github.com/slavpilus/mcp
cd mcp
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt

Step 2: Configure Claude Desktop

macOS: Edit ~/Library/Application Support/Claude/claude_desktop_config.json

Windows: Edit %APPDATA%\Claude\claude_desktop_config.json

{
  "mcpServers": {
    "enneagora": {
      "command": "python",
      "args": ["/full/path/to/mcp/main_stdio.py"],
      "env": {}
    }
  }
}

‚ö†Ô∏è Important: Use the full absolute path to main_stdio.py and ensure your Python environment has all dependencies installed.

Step 3: Start Local Server

The STDIO version will be automatically started by Claude Desktop. To test manually:

python main_stdio.py

üîß Available Tools (14 Total)

Enneagora provides comprehensive e-commerce support with these tools:

get_order_status

Get detailed status for a specific order including tracking information

Parameters:

  • order_id (string) - Order identifier (e.g., "ORD-1001")
  • customer_id (string, optional) - Customer validation

cancel_order

Cancel an order with validation checks

Parameters:

  • order_id (string) - Order to cancel
  • reason (string, optional) - Cancellation reason
  • customer_id (string, optional) - Customer validation

process_return

Handle return requests with prepaid labels

Parameters:

  • order_id (string) - Order to return
  • item_ids (array, optional) - Specific items
  • reason (string, optional) - Return reason

track_package

Track package delivery with real-time updates

Parameters:

  • order_id (string) - Order to track
  • customer_id (string, optional) - Customer validation

get_support_info

General support information and guidance

Parameters:

  • topic (string, optional) - Support topic
  • customer_id (string, optional) - Customer context

get_return_policy

Detailed return policies by product category

Parameters:

  • product_category (string, optional) - Category-specific policy

get_shipping_info

Shipping rates, delivery times, and options

Parameters:

  • destination_country (string, optional) - International rates
  • order_value (number, optional) - Free shipping check

get_contact_information

Contact details by issue type and urgency

Parameters:

  • issue_type (string, optional) - Type of issue
  • urgency (string, optional) - Urgency level

get_size_guide

Size charts for clothing, shoes, and accessories

Parameters:

  • product_type (string) - Product type
  • brand (string, optional) - Brand-specific sizing

get_warranty_information

Warranty coverage and claims processing

Parameters:

  • product_category (string) - Product category
  • purchase_date (string, optional) - Warranty status check

get_payment_information

Payment methods, billing help, and troubleshooting

Parameters:

  • inquiry_type (string, optional) - Payment inquiry type

get_account_help

Account troubleshooting and login assistance

Parameters:

  • issue_type (string) - Account issue type

get_loyalty_program_info

Rewards program details and member benefits

Parameters:

  • inquiry_type (string, optional) - Program inquiry type

get_product_care_info

Care instructions and maintenance guidance

Parameters:

  • product_category (string) - Product category
  • material (string, optional) - Specific material

üß™ Dynamic Testing Data

Enneagora features a dynamic mock data system where order behavior is determined by the order ID pattern. You can test various scenarios using these special order IDs:

Order Status Patterns

Use these order ID suffixes to test different order states:

  • ORD-XXXX-D - Delivered orders (e.g., ORD-1001-D)
  • ORD-XXXX-S - Shipped orders with tracking (e.g., ORD-1002-S)
  • ORD-XXXX-T - In Transit orders with location updates (e.g., ORD-1003-T)
  • ORD-XXXX-P - Processing orders (e.g., ORD-1004-P)
  • ORD-XXXX-R - Ready for Pickup (e.g., ORD-1005-R)
  • ORD-XXXX-C - Cancelled orders (e.g., ORD-1006-C)
  • ORD-XXXX-F - Failed/Problem orders (e.g., ORD-1007-F)
  • ORD-XXXX-E - Error/Not Found (e.g., ORD-1008-E)
  • ORD-XXXX - Pending orders (no suffix, e.g., ORD-1009)

Special Test Cases

Try these specific scenarios:

  • ORD-ERROR - Always returns "not found"
  • ORD-1001-F - Failed order with "lost package" tracking
  • ORD-2002-F - Cannot be cancelled (simulates payment issues)
  • ORD-3003-D - Delivered order with full tracking history
